# Security Policy

## 支持范围

当前维护分支为 `main`。本项目定位为企业内部 POC、实习项目投递和面试演示，不承诺生产 SaaS 级别的安全响应 SLA。

## 报告漏洞

如果你发现安全问题，请通过 GitHub 联系仓库所有者，或提交不包含敏感细节的 issue。公开 issue 中请不要粘贴真实 API key、客户数据、私有业务文档、可直接复现利用的 payload 或内部地址。

报告时建议包含：

- 受影响的模块或接口。
- 影响范围和风险等级判断。
- 最小化的复现步骤，必要时请用脱敏样例。
- 你期望的修复方向或缓解建议。

## 敏感信息处理

本仓库不应提交 `.env`、真实客户数据、生产 API key、真实工单、私有知识库原文或备份文件。示例配置请放在 `.env.example`，示例数据应使用合成内容。

如果误提交了敏感信息，应立即撤销相关凭证，并在清理 Git 历史后重新推送。
